# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.3.3] - 2026-05-05

### Fixed

- **BUG-1**: Fixed dead type expression in `SqlServer#rows_to_be_inserted`.
- **BUG-2**: Fixed scaffold output losing colon prefix on long column names (`>= 40` chars).
- **BUG-3**: Added descriptive error message for unknown conditional methods in `ConfigApplicator`.
- Fixed MySQL string parser regex backtracking risk by using a character-class-based approach.
- Fixed Postgres `rows_to_be_inserted` potential `IndexError` when row is empty.
- Fixed `config_scaffold_generator` writing null byte (`\0`) to scaffold output.
- Fixed fragile type detection in `formatted_line` (`definition.to_s[0]`) by using `is_a?` checks.
- Guarded against empty-buffer crash in `config_scaffold_generator` when all tables are `:keep`/`:truncate`.
- Fixed typos: `begginning` → `beginning`, `statments` → `statements`, `es not` → `is not`.

### Changed

- **Security**: Upgraded email hash from `Digest::MD5` (5 hex chars, ~1M entropy) to `Digest::SHA256` (10 hex chars, ~1.1 trillion entropy).
- **Idiomatic Crystal**: Replaced manual `rand` arithmetic with `rand(range)`, `(rand * chars.size).to_i` with `chars.sample`, `String.new` with `""`, and string concatenation with interpolation.
- **Performance**: `random_string` now uses `String.build` instead of `+=` in a loop (O(n) vs O(n²)).
- Replaced 4 chained `gsub` calls in SqlServer with efficient `lstrip`/`rstrip`/`sub`.
- Removed redundant `is_a?(Hash)` check in `ConfigApplicator#apply_table_config`.
- Removed dead `fail_on_unspecified_columns?` manual method (already generated by `property?`).
- Removed unnecessary `# encoding: UTF-8` comment from `mysql.cr`.
- `missing_columns.size == 0` → `missing_columns.empty?`.

### Type Annotations

- Added explicit parameter and return types to all abstract methods in `Base` and every concrete implementation across `Mysql`, `Postgres`, and `SqlServer`.
- Added explicit types to `Triki#config` (`ConfigHash`) and `Triki#scaffolded_tables` (`Hash(String, Int32)`).
- Narrowed `missing_column_list` return from bare `Array` to `Array(String)`.
- Narrowed `make_conditional_method` return from bare `Proc` to `Proc(RowAsHash, Bool)`.

### Documentation

- Added doc comments to all public APIs: `Triki`, `ConfigApplicator`, `Mysql`, `Postgres`, `SqlServer`, `FakerInterface`, `EnglishDictionary`.
- Hidden internal infrastructure (`Base`, `InsertStatementParser`, `ConfigScaffoldGenerator`, `ConfigParser`, aliases, helper methods) from generated docs using `:nodoc:`.

### Tests

- Added `random_integer` range test (verifies `rand(range)` produces values within bounds).
- Added `random_string` charset and length tests.
- Added `EnglishDictionary` sentence word-count test (3–7 words).
- Added empty scaffold buffer test (all `:keep`/`:truncate` tables don't crash).
- Added `fail_on_unspecified_columns?` getter test.
- Added `clean_quotes`, `clean_bad_whitespace`, and `make_conditional_method` unit tests.

## [0.3.2] - 2024-01-15

### Added

- Initial support for MariaDB dumps.
- `AGENTS.md` developer guide.

### Fixed

- Various parser edge cases for quoted strings in MySQL dumps.

## [0.3.1] - 2023-11-20

### Fixed

- Compatibility fixes for Crystal 1.10+.

## [0.3.0] - 2023-09-10

### Added

- SQL Server support.
- `:secondary_address` obfuscation type.

## [0.2.5] - 2023-06-01

### Fixed

- Fixed handling of `\N` NULL values in Postgres dumps.

## [0.2.4] - 2023-03-15

### Added

- `:skip_regexes` option for conditional skipping.

## [0.2.3] - 2023-01-10

### Fixed

- Fixed INSERT IGNORE parsing for MySQL.

## [0.2.2] - 2022-11-20

### Added

- `:like_english` obfuscation type using word-frequency lists.

## [0.2.1] - 2022-09-01

### Fixed

- Compatibility with Crystal 1.5+.

## [0.2.0] - 2022-07-15

### Added

- PostgreSQL `COPY ... FROM stdin` support.

## [0.1.1] - 2022-05-01

### Fixed

- Fixed hex literal parsing in MySQL dumps.

## [0.1.0] - 2022-03-15

### Added

- Initial release with MySQL support.
- Core obfuscation types: `:email`, `:name`, `:string`, `:integer`, `:fixed`, `:null`, `:keep`.
